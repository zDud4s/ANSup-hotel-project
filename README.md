# Hotel Booking Demand — Clustering Study

**Course:** Unsupervised Learning · 2025/2026 · FCT NOVA
**Dataset:** Hotel Booking Demand (António et al., 2019)

---

## Segmentation Question

> **What recurring booking-behaviour profiles emerge from hotel reservation records, using only information available at the moment of booking confirmation?**

Unit of analysis: one row = one booking record. Conclusions are expressed
as recurring booking behaviours, not as definitive customer types.

Index time = booking confirmation. Every clustering input must be available
at or before that point; anything updated afterwards is reserved for
post-hoc profiling only. Feature governance is centralised in
[`src/preprocessing/feature_config.py`](src/preprocessing/feature_config.py).

See [pitch/PITCH.md](pitch/PITCH.md) for the full problem statement, dataset
documentation, evaluation protocol, success criteria, and risks/ethics.
